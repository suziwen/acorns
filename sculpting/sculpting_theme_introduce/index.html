Sculpting 主题配置说明文档

Sculpting 主题是一款基本 Gatsbyjs 开发并使用小书匠导出的 zip 做为数据源的博客主题.

一、主要特性

  • 基于 Gatsby
  • 使用 Reactjs
  • 基于 Theme UI 主题配置
  • 支持代码块语法高亮
  • 响应式设计
  • 标签页
  • 归档页
  • 搜索页
  • rss

二、快速搭建一个自己的 Sculpting 博客

  1. 直接克隆一份 suziwen/gatsby-action-sculpting


克隆代码

  1. 访问 https://github.com/{yourname}/gatsby-action-sculpting/actions, 激活 action 功能
  2. 到 https://github.com/settings/tokens/new, 创建一个 GitHub Personal Access Token, token 范围选择 repo


创建 Token


成功创建 token

  1. 到 https://github.com/{yourname}/gatsby-action-sculpting/settings/secrets 页面, 添加一个 secret, 名称为 ACCESS_TOKEN , 值为上一个步骤生成的 token


创建 ACCESS_TOKEN 环境变量

  1. 使用小书匠编辑器,写一篇文章,然后导出为 zip 文件
  2. 访问 https://github.com/{yourname}/gatsby-action-sculpting/tree/master/posts, 点击 Upload Files , 将上一步骤导出的 zip 文件上传到这里


上传 zip

  1. 等 github action 自动编译完成后,就可以访问页面 http://{yourname}.github.io/gatsby-action-sculpting


action status

注:

需要将 {yourname} 替换成你自己的 github 帐号名称

三、开始使用

本博客使用 Gatsby 进行开发,在开始使用 Sculpting 时,建议先了解下一些 Gatsby 的开发流程.

Gatsby 支持两种创建博客方式,一种是全新的安装模式,一种是在现有的 Gatsby 博客上进行扩展.

1. 全新模式安装

  1. 1gatsby new suziwen/gatsby-starter-sculpting 

2. 扩展模式安装

如果你已经有了一个 Gatsby 的静态网站,想在该网站的基础上安装 sculpting 主题,可以使用这种方式安装.

  1. 安装 gatsby-theme-sculpting 组件
  1. 1npm install `@suziwen/gatsby-theme-sculpting` 
  1. 在配置文件 gatsby-config.js 里注册安装的组件
gatsby-config.jsjavascript    3行
  1. 1module.exports = { 
  2. 2 plugins: [ 'gatsby-theme-sculpting'] 
  3. 3} 

四、配置

1. 插件支持的参数

可以使用下面的形式,对 gatsby-theme-sculpting 主题进行参数设置

gatsby-config.jsjavascript    22行
  1. 1module.exports = { 
  2. 2 plugins: [ 
  3. 3 { 
  4. 4 resolve: 'gatsby-theme-sculpting', 
  5. 5 options: { 
  6. 6 basePath: '/', 
  7. 7 tagsPath: 'tags', 
  8. 8 archivesPath: 'archives', 
  9. 9 contentPath: 'posts', 
  10. 10 disqus: { // 需要使用时再配置这个节点 
  11. 11 shortname: '' 
  12. 12 }, 
  13. 13 gitalk: { // 需要使用时再配置这个节点 
  14. 14 clientID: '', 
  15. 15 clientSecret: '', 
  16. 16 repo: '', 
  17. 17 owner: '', 
  18. 18 admin: [] 
  19. 19 } 
  20. 20 } 
  21. 21 ] 
  22. 22} 

1-1. BasePath

生成博客的基础路径,默认为值为 /

1-2. tagsPath

标签的基础访问路径,默认值为 tags

1-3. archivesPath

归档的基础访问路径,默认值为 archives

1-4. contentPath

指定放置小书匠 zip 文件的位置(相对路径时,以博客的安装目录为起点算起),默认值为 posts

1-5. 评论

目前支持 disqus 和 gitalk.

当同时配置了 disqus 和 gitalk 时,系统直接使用 gitalk.

2. 通用的 sitemetadata 配置

可以在 gatsby-config.js 里的 sitemetadata 节点,配置一些网站的基本信息.

gatsby-config.jsjavascript    11行
  1. 1module.exports = { 
  2. 2 siteMetadata: { 
  3. 3 title: '小书匠', // 网站标题, 页面左上角的文字 
  4. 4 siteUrl: 'http://www.xiaoshujiang.com', // 网站地址 
  5. 5 description: '', // 对网站的一些介绍 
  6. 6 idiom: '', // 谚语,可以为空,页面左下角的文字 
  7. 7 author: '', // 作者, 页面右下角 copyright 上作者信息. 
  8. 8 }, 
  9. 9 plugins: [...] 
  10. 10} 
  11. 11 

五、文章元数据支持

1. title

文章标题

2. createDate

文章创建时间,如果不指定,系统直接使用内置的创建时间

3. slug

用于生成文章的访问路径,如果不指定,直接使用 title 做为访问路径.

4. tags

标签,多个标签以逗号分开,也可以使用 yaml 的数组定义多个标签

5. excerpt

文章摘要

六、定制

想要对主题进行微调的,需要使用到 Gatsby 的 shadowing component 功能,进行继承修改,详细教程可以参考 Gatsby 官方教程.

当然你也可以直接在主题上进行修改,然后重新发布一个新的主题.
